## Что можно сохранить

### Для `hr/hr-specialist/`

- **Процесс «Импорт Excel → нормализация → Проверка записей → утверждение»** — ядро кадрового контура.
- Критерии **когда утверждать / отклонить** запись (образование, сертификат, даты, дубликаты).
- **Пошаговый сценарий review batch** (фильтр, Dry Run, Promote, «Изменения реестра»).
- **Объяснение Cancel / Annul / Archive** для кадровиков — сильный текст, понятный бизнес-язык.
- **Матрица полномочий** (оператор → начальник ОК → руководитель кадров → архивариус).
- **Блок «Производственные приказы»** (OO intake) — основа для `UG-HR-001-orders/`.
- Часть **скриншотов импорта/review** (после сверки с текущим UI Personnel).

### Для `hr/hr-head/`

- Раздел про **аннулирование зарегистрированных приказов** и governance.
- Объяснение, почему sysadmin ≠ кадровые полномочия.

### Для `docs-work/notes/hr/` (не UG)

- ADR-039/040/043, Lifecycle Run, Effective Person Viewer — **техническая документация**, не end-user.

## Чего не хватает

- Отдельные UG по **Personnel Orders** (создание, подписание, void) с привязкой к UI `directory/personnel`.
- **Operational Orders** — только intake; нет ready-for-signature / sign (если нужно кадровику).
- Единый **glossary** терминов UI (Персонал, Проверка записей, Изменения реестра).
- Скриншоты **без** internal/debug экранов для hr-head.
- Раздел для **hr-head**: контроль работы специалистов, не операционные клики.
