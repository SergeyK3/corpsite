# Анонимизированные паттерны исполнения (OP-RES-004)

Иллюстративные формулировки без ФИО и без привязки к конкретным документам.

---

## Pattern A — Командировка (S_TRAVEL)

```text
1. [Направить] [должность/ФИО] в [город] [даты] для [цель мероприятия].
2. [Расходы] за счёт [источник финансирования].
3. [Сохранить] место работы и средний заработок на период командировки.
4. [Контроль] оставляю за собой.                    ← Control Obligation (order-level)
5. Приказ вступает в силу со дня подписания.        ← meta-item, не execution
Основание: [заявление сотрудника].
```

**Разбор:** пункты 1–3 — Execution Obligations; пункт 4 — Control Obligation; пункт 5 — document effect.

---

## Pattern B — Комиссия (S_COMMISSION)

```text
1. Создать комиссию по [предмет].
   Председатель: [должность].
   Члены: [должность]; [должность]; …
2. Контроль за исполнением возложить на [должность главбуха].
```

**Разбор:** пункт 1 совмещает создание объекта (Commission as Managed Object) и назначение состава (Commission as Party). Пункт 2 — Control Obligation, не исполнение комиссии.

---

## Pattern C — Дисциплина (S_DISCIPLINE)

```text
1. [Назначить санкцию] [сотруднику] в виде [тип взыскания].
2. [Заведующий отделением] взять на контроль работу сотрудников отделения.  ← embedded control
3. [Зам. директора] усилить контроль за трудовой дисциплиной.               ← directional control
4. [HR] в течение 3 рабочих дней ознакомить [сотрудника] с приказом.       ← separate obligation + deadline
```

**Разбор:** один участник (заведующий) — контролёр в п.2; зам. директор — другой контроль в п.3; HR — исполнитель п.4. Роли не взаимоисключающие между разными лицами; один человек может иметь несколько ролей в разных пунктах.

---

## Pattern D — Клинический процесс (S_CLINICAL)

```text
1. [Заведующему отделения X] обеспечить работу [специалистов] в режиме [время].
2. [Заведующему отделения Y] организовать [кабинет/услугу].
…
N-1. Контроль возложить на [зам. директора по лечебной работе].
N.   Приказ в силу со дня подписания.
```

**Разбор:** параллельные Execution Obligations (п.1, п.2, …); общий контроль на весь приказ в п.N-1.

---

## Pattern E — Выделение средств (S_FUNDS)

```text
1. [Перечислить] денежные средства [сумма] на [цель/счёт].
2. [Ответственному] по окончании [мероприятия] представить авансовый отчёт с подтверждающими документами.
3. Контроль оставляю за собой.
```

**Разбор:** последовательная зависимость п.2 от п.1 (событие-триггер «по окончании»). Expected Result п.1 = transfer; п.2 = report (Execution Evidence).

---

## Pattern F — Регламент с приложением (S_REGULATION / S_PHARMA)

```text
1. Утвердить [положение/перечень/состав] (согласно приложению 1).
2. [Ответственным лицам] обеспечить исполнение настоящего приказа.
3. Контроль возложить на [должность].
```

**Разбор:** приложение содержит детализацию обязательств, не дублированную в тексте. Attachment-driven execution.

---

## Pattern G — Подпункты (nested)

```text
1. Ввести три вида пропуска:
   1.1. Служебный — для [категория];
   1.2. Персональный — для [категория];
   1.3. Посетительский — для [категория].
2. Установить порядок выдачи пропусков: …
```

**Разбор:** Execution Unit = подпункт 1.1–1.3 как дочерние обязательства единого пункта 1.

## Role notation (анонимизировано)

| Код в матрице | Значение |
|---|---|
| `role_head_of_unit` | Заведующий подразделением (должность) |
| `role_deputy_director` | Заместитель директора |
| `director_self` | Директор — контроль «оставляю за собой» |
| `role_chief_accountant` | Главный бухгалтер |
| `named_employee` | Конкретный сотрудник (ФИО в источнике) |
| `commission_members` | Коллективный орган |

---

## Control vs Responsibility (не смешивать)

| Формулировка | Тип |
|---|---|
| «Контроль возложить на…» | Control Obligation |
| «Ответственность возложить на…» | Blended (control + accountability) |
| «Взять на контроль работу…» | Embedded ongoing control duty |
| «Обеспечить исполнение…» | Execution Obligation (не контроль) |
| «Ознакомить с приказом…» | Separate execution + evidence |
