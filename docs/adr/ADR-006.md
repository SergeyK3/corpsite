# ADR-006: Roadmap — переход от Events/Notifications к системе личных кабинетов

## Статус
Принято

## Дата
2026-01-12

## Контекст

На предыдущем этапе (ADR-005) была стабилизирована подсистема Events / Notifications:
- подтверждён e2e-сценарий доставки событий (report → approve / reject → report → approve);
- зафиксирован cursor-based polling;
- согласованы UX-тексты Telegram-интерфейса;
- проведена проверка ролевой логики на уровне backend + bot.

На текущий момент система обладает:
- устойчивым API задач;
- событийной моделью (audit_log / events);
- Telegram-интерфейсом как вспомогательным каналом.

Следующий логический этап развития системы — **основной рабочий интерфейс пользователей**:
система личных кабинетов (ЛК), зависящих от роли пользователя.

## Принципиальное решение

1. **Не выполнять сейчас UX-шлифовку мелких вариантов (A/B/C)** в Telegram и Events.
2. Зафиксировать Events/Notifications как стабильный фундамент.
3. Перейти к **следующему крупному этапу roadmap**:
   проектированию и реализации системы личных кабинетов.

Telegram-интерфейс рассматривается далее как:
- канал уведомлений;
- fallback / быстрый доступ;
- вспомогательный инструмент, но не основной UI.
