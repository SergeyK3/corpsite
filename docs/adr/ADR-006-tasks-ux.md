# ADR-006 — UX-контракт команд /tasks и событий (Events)

## Статус
**Принято (Final)**

## Дата
2026-01-11

## Контекст
В рамках блока **Tasks / Events** зафиксированы пользовательские тексты и поведение Telegram-бота
для команд управления задачами.  
Цель — обеспечить единый, предсказуемый UX на всех клиентах (ПК, планшет, телефон),
без расхождений между backend и ботом.

Данный документ является **UX-контрактом**.  
Изменения допускаются только через новый ADR.

---

## Общие UX-принципы (обязательные)

1. **Одно действие → один ответ**
2. **Всегда сообщается итоговый статус задачи**
3. **Комментарий пользователя возвращается в ответе**
4. **Короткие, недвусмысленные формулировки**
5. **Идентичное поведение на ПК и планшете**
6. **Backend — источник истины, бот не интерпретирует бизнес-логику**
