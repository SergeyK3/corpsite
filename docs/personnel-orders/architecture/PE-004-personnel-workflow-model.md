# PE-004 Personnel Workflow Model

**Статус:** Draft  
**Версия:** 0.1  
**Зависимости:** PE-001, PE-002, PE-003

---

# 1. Назначение

Документ определяет модель жизненного цикла кадрового события в кадровом контуре Corpsite.

Ключевой принцип: **кадровое событие и приказ имеют разные жизненные циклы**.

Кадровое событие отвечает на вопрос:

> Что меняется в кадровом состоянии сотрудника?

Приказ отвечает на вопрос:

> Каким юридическим документом это изменение оформлено?

---

# 2. Базовая цепочка

```text
Personnel Event
    → Change Set
    → Personnel State
```

Кадровое событие порождает набор изменений.  
Набор изменений изменяет расчетное кадровое состояние сотрудника.

---

# 3. Жизненный цикл кадрового события

```text
Draft
    ↓
Prepared
    ↓
Approved
    ↓
Active
    ↓
Completed
    ↓
Archived
```

## 3.1 Draft

Событие создано, но еще не готово к оформлению.

Примеры:

- проект приема;
- проект отпуска;
- проект перевода;
- проект изменения ставки.

## 3.2 Prepared

Событие заполнено кадровым специалистом и готово к согласованию или включению в приказ.

## 3.3 Approved

Событие согласовано как бизнес-изменение.

На этом этапе еще может не быть подписанного приказа.

## 3.4 Active

Событие вступило в силу и изменило Personnel State.

## 3.5 Completed

Событие завершилось по сроку действия.

Например:

- завершилась командировка;
- завершилось временное исполнение обязанностей;
- завершилась доплата на период замещения.

## 3.6 Archived

Событие не активно, но сохраняется в истории.

---

# 4. Альтернативные статусы

## Cancelled

Событие отменено до вступления в силу или после регистрации отменяющим документом.

## Reversed

Событие компенсировано обратным кадровым событием.

## Superseded

Событие заменено более поздним событием.

---

# 5. Связанные события

Некоторые кадровые события создают связанные события.

Пример отпуска:

```text
Leave Event
    ├── Temporary Assignment Event
    └── Allowance Event
```

Пример длительной командировки:

```text
Business Trip Event
    ├── Temporary Assignment Event
    └── Allowance Event
```

Связанные события являются самостоятельными кадровыми событиями, но имеют ссылку на исходное событие.

---

# 6. Визы и согласование

Согласование не является кадровым событием.

Визы относятся к workflow проекта приказа или проекта кадрового события.

Для разных типов событий маршрут согласования может отличаться.

Примеры:

| Событие | Типовые согласующие |
|---|---|
| Отпуск | непосредственный руководитель, HR |
| Перевод | текущий руководитель, новый руководитель, HR |
| Доплата | руководитель, финансы, HR |
| Увольнение | руководитель, HR, юрист/расчетчик при необходимости |

---

# 7. Правила исполнения

1. Personnel State не должен изменяться от черновика.
2. Personnel State изменяется только после наступления EffectiveFrom у утвержденного события.
3. Событие с EffectiveTo должно автоматически завершаться после окончания периода.
4. Отмена события должна быть трассируемой.
5. Любое изменение должно быть связано с Change Set.

---

# 8. Вывод

Personnel Workflow Model отделяет бизнес-жизненный цикл кадрового события от жизненного цикла приказа и документов.

Это позволяет:

- оформлять несколько событий одним приказом;
- хранить события без зависимости от DOCX/PDF;
- поддерживать связанные события;
- автоматически рассчитывать актуальное состояние сотрудника.
