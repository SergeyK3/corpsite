# PO-PRINT-001 — Печатная форма кадрового приказа

**Статус:** Implemented (MVP print UI + official PDF)  
**WP:** WP-PO-PRINT-002 / WP-PO-PRINT-003 / WP-PO-PDF-001  
**Код:** `corpsite-ui/app/directory/personnel/**/print/**`, `**/pdf/**`, `_lib/personnelOrderPrint*`, `_lib/personnelOrderPdf*`

---

## 1. Назначение

Экранный предпросмотр и **официальный PDF** кадрового приказа в режимах:

- `kk`
- `ru`
- `kk-ru`

| Канал | Маршрут | Роль |
|---|---|---|
| HTML preview | `/directory/personnel/orders/{id}/print?language=` | экранный предпросмотр; браузерная печать HTML |
| Official PDF | `/directory/personnel/orders/{id}/pdf?language=` | серверный PDF без браузерных колонтитулов |

Маршруты, выбор языка и система item-renderer’ов сохраняются; печатная форма — **проекция** данных приказа, не отдельный SoT.  
Официальный PDF-движок: [`PO-PDF-001-official-pdf-engine.md`](./PO-PDF-001-official-pdf-engine.md).

### Типографика документа (PO-PRINT-004)

Локально на `.personnel-order-print-document` (не на весь UI):

- `font-family: "Times New Roman", Times, serif;`
- `font-size: 14pt;`
- `line-height: 1.35;`

Исключения: тип документа 16 pt; подпись под линией ознакомления 11 pt.  
Панель «Назад | Язык | PDF / Печать | Печать HTML» сохраняет UI-шрифт и скрывается через `print:hidden`.  
Подсказка про отключение браузерных колонтитулов относится **только** к HTML browser print; основной сценарий — PDF.

## 2. Статусы печатной формы

Системные коды статуса **не** выводятся пользователю как есть.

Человекоориентированная шкала:

```text
ПРОЕКТ
    ↓
НА ПОДПИСЬ
    ↓
официальный подписанный приказ
    ↓
АННУЛИРОВАН
```

| Системный статус | Печатная отметка (watermark) | Примечание |
|---|---|---|
| `DRAFT` | **ПРОЕКТ** / **ЖОБА** | Черновик; не официальный документ |
| `READY_FOR_SIGNATURE` | **НА ПОДПИСЬ** / **ҚОЛ ҚОЮҒА** | Документ готов к подписанию; **не** «Макет» / «Не подписан» |
| `SIGNED` | нет | Официальный подписанный приказ (без имитации ЭП, если подпись не хранится) |
| `REGISTERED` | нет | Официальный; при необходимости позже — служебная пометка «Зарегистрирован» |
| `VOIDED` | **АННУЛИРОВАН** / **КҮШІ ЖОЙЫЛҒАН** | Явная отмена силы документа |

### Правила watermark

- Диагональный, полупрозрачный.
- Виден на экране и при печати (`print-color-adjust`).
- Не перекрывает читаемость основного текста.
- Автоматически скрывается для `SIGNED` и `REGISTERED`.

### Запрещённые формулировки для `READY_FOR_SIGNATURE`

Не использовать:

- «Макет приказа»
- «Не подписан»

Для кадрового процесса корректна формулировка **«НА ПОДПИСЬ»**.

## 3. Архитектурные границы

Печатный контур **не** включает:

- ЭЦП / корпоративную ЭП;
- хранение подписанного PDF;
- электронное ознакомление;
- удаление приказов.

Концепции:

- официальный PDF (динамический MVP) — [`PO-PDF-001-official-pdf-engine.md`](./PO-PDF-001-official-pdf-engine.md);
- подписание — [`PO-SIGN-001-signing-concept.md`](../PO-SIGN-001-signing-concept.md);
- удаление / void — [`PO-LIFECYCLE-002-delete-and-void-policy.md`](../PO-LIFECYCLE-002-delete-and-void-policy.md).

---

## 4. Принцип

Неподписанный приказ на печати = **проект** или документ **на подпись**.  
Подписанный / зарегистрированный = **официальный** документ без watermark проекта.  
Способ фиксации подписи проектируется отдельно (PO-SIGN-001).
