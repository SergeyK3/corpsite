# Решение проблемы двуязычия

Первичным документом организации является приказ на казахском языке.

Карточка приказа должна объединять:

- DOCX (казахский);
- PDF (подписанный оригинал);
- DOCX (русский);
- PDF (русский перевод).

Все версии относятся к одному приказу и имеют единый регистрационный номер.

# Предметная модель модуля Personnel Orders

## Общие положения

Анализ пилотного архива показал, что центральной сущностью кадрового контура является не приказ и не файл документа, а кадровое событие (Personnel Event).

Приказ является юридическим оформлением одного или нескольких кадровых событий.

Электронные документы (DOCX, PDF, сканы) являются файловым представлением приказа.

Базовая модель:

```text
Personnel Event
    ← Order Item
        ← Personnel Order
            ← Order Document
```

## Основные сущности

### Employee

Сотрудник — центральная сущность кадрового контура.

### Personnel Event

Изменение состояния сотрудника.

Типовые события:

- прием;
- перевод;
- увольнение;
- изменение ставки;
- совмещение;
- изменение оплаты;
- изменение образования;
- изменение квалификации;
- изменение персональных данных;
- выход из отпуска.

### Personnel Order

Юридическое оформление одного или нескольких кадровых событий.

### Order Item

Пункт приказа.

Каждый пункт описывает одно кадровое событие одного сотрудника.

### Order Document

Файловое представление приказа.

Поддерживаемые типы:

- DOCX KZ
- PDF KZ (подписанный)
- DOCX RU
- PDF RU
- Scan

### Personnel Report

Самостоятельная сущность.

Кадровые отчеты не являются приказами и должны храниться отдельно в разделе Personnel Reports.

# Результаты анализа пилотного архива

Период анализа: первое полугодие 2026 года.

Исследовано:

- 43 документа Word;
- выявлены кадровые приказы и кадровые отчеты.

Основные выводы:

1. Приказ не является первичной сущностью кадрового контура.
2. Один приказ может содержать несколько кадровых событий.
3. Одно кадровое событие относится к одному сотруднику.
4. Кадровые отчеты должны храниться отдельно от приказов.
5. Электронная модель должна строиться вокруг Personnel Event.

---

# Основной архитектурный принцип

В модуле Personnel Orders первичной сущностью является кадровое событие.

Приказ представляет собой юридическую регистрацию кадровых событий.

Файлы DOCX/PDF являются представлением приказа.

Такой подход обеспечивает:

- полную историю кадровых изменений;
- поддержку нескольких событий в одном приказе;
- автоматическое формирование двуязычных документов;
- построение аналитики по кадровым событиям без анализа текста приказов.
