# PO-003 Модель данных модуля приказов по личному составу

**Статус:** Черновик  
**Версия:** 0.1

## Назначение

Документ определяет концептуальную модель данных модуля «Приказы по личному составу».

Архитектурная цепочка:

Personnel Event
→ Change Set
→ Order Item
→ Personnel Order
→ Order Document

---

# Основные сущности

## Personnel Event

Кадровое событие, изменяющее кадровое состояние сотрудника.

Основные поля:

- идентификатор;
- сотрудник;
- тип события;
- подтип;
- статус;
- дата начала;
- дата окончания;
- родительское событие;
- связанное событие;
- основание.

---

## Personnel Change Set

Набор изменений, принадлежащий одному кадровому событию.

---

## Personnel Change

Атомарное изменение.

Основные поля:

- изменяемый атрибут;
- прежнее значение;
- новое значение;
- период действия.

Рекомендуется хранить одновременно текстовое и структурированное (JSON) представление значений.

---

## Personnel Order

Приказ по личному составу.

Основные реквизиты:

- номер;
- дата;
- вид приказа;
- статус;
- язык оригинала;
- подписант;
- исполнитель.

---

## Order Item

Пункт приказа.

Связывает одного сотрудника и одно кадровое событие с приказом.

---

## Order Document

Файловые представления приказа:

- DOCX (KZ);
- PDF (KZ, подписанный);
- DOCX (RU);
- PDF (RU);
- скан.

---

## Order Approval

Маршрут согласования.

Хранит:

- согласующего;
- роль;
- статус;
- дату;
- комментарий.

---

# Связанные кадровые события

Пример:

Отпуск
→ Временное исполнение обязанностей
→ Доплата

Связь рекомендуется реализовать через идентификаторы родительского/связанного события либо отдельную таблицу связей.

---

# Рекомендуемые статусы

### Кадровое событие

- черновик;
- согласовано;
- активно;
- завершено;
- отменено.

### Приказ

- черновик;
- подготовлен;
- согласование;
- подписан;
- зарегистрирован;
- исполнен;
- архив;
- отменен.

### Согласование

- ожидает;
- согласовано;
- отклонено;
- пропущено.
